LIGMA
UI/UX Design Specification
Virtual Collaboration Workspace — DevDay &apos;26 Hackathon

1. Design Philosophy &amp; Principles
1.1 Core Design Tenets
Design Philosophy
LIGMA&apos;s UI philosophy is &apos;canvas-first, clutter-never.&apos; The canvas occupies 100% of the viewport. All secondary panels (Task Board, Event Log) are overlays or collapsible side panels that never diminish the creative space. Every control has a purpose; no decoration without function.

Principle 1 — Spatial Clarity
The canvas is the product. UI chrome is minimal and retreats when not in use. Toolbar collapses to icons only. Panels slide in from edges. Cursor focus is always on the canvas.
Principle 2 — Real-Time Trust
Users must always know who else is present and what they are doing. Cursors are labeled, role badges are visible, and connection status is always surfaced. There is no ambiguity about whether a colleague is in the room.
Principle 3 — Intent Surfaces Itself
The AI classification is visible but unobtrusive. Tags appear as small color-coded badges on nodes, not modal interruptions. The Task Board is a natural extension of the canvas, not a separate application.
Principle 4 — Zero Dead Ends
Every action is recoverable. Undo is always available (Ctrl+Z). Deleting a node does not destroy its Task Board entry. The event log is always accessible. There are no permanent destructive actions without confirmation.

3.5 Presence Cursors (Canvas Overlay)
Cursor Rendering
Each remote user&apos;s cursor is rendered as an SVG arrow cursor in their assigned color
Cursor label: small pill attached to cursor tip showing user&apos;s first name (max 12 chars, truncated with ellipsis)
Label background: user&apos;s cursor color at 90% opacity; white text
Cursor animation: position interpolated smoothly using lerp (linear interpolation) at 60fps to prevent jitter from network updates arriving at 50ms intervals
Idle timeout: if a cursor has not moved for 30 seconds, label fades to 40% opacity and a &apos;(idle)&apos; suffix appears
Disconnect: cursor fades out over 1 second and is removed from canvas
Presence Avatars Strip (Top-Right)
Shows 24px circular avatar chips for each connected user, with initials + color
Tooltip on hover: full name + role badge
Overflow: if &gt;5 users, shows &apos;+N more&apos; chip that expands to list on click
Indicator dot: green = active, yellow = idle (&gt;30s no input), gray = away/disconnected

3.6 Connection Status Indicator
Bottom-right corner of canvas, 12px, always visible

5. User Flows
5.1 Flow 1 — Create and Join a Room
User navigates to /login
User enters display name and selects role (Lead)
User clicks &apos;Create New Room&apos;
System generates unique roomId, redirects to /room/:roomId
Canvas workspace loads; empty canvas shown; user sees their own cursor
User copies URL from browser address bar
Second user navigates to URL, enters display name, selects role (Contributor)
Second user joins; Lead sees second user&apos;s cursor appear with name label
Second user sees existing canvas state (empty initially) and first user&apos;s cursor

5.2 Flow 2 — Collaborative Brainstorm with AI Task Extraction
Contributor double-clicks canvas → sticky note created at cursor position
Contributor types &apos;Assign to Alex: Implement auth flow by Friday&apos;
After 1.5s debounce: client sends text to AI classifier via API
All users see &apos;Analyzing...&apos; pulsing badge on the sticky note
AI responds: ACTION_ITEM — badge updates to red &apos;ACTION ITEM&apos; badge on all screens
Task Board panel (if open) shows new entry: task text, author, timestamp, canvas link
Lead clicks Task Board entry → canvas pans and zooms to center the sticky note
Lead right-clicks sticky note → selects &apos;Lock to Lead&apos;
Lock icon appears on sticky note for all users
Contributor attempts to edit locked sticky note → tooltip &apos;This node is locked&apos; shown

5.3 Flow 3 — Reconnection after Drop
User&apos;s network drops → client detects missing pong within 10 seconds
Connection status changes to &apos;Reconnecting (1/5)...&apos; (yellow indicator)
Other users&apos; cursors stop moving; disconnected user&apos;s cursor fades out for others
Client attempts reconnect with exponential backoff
On successful reconnect: client sends { type: &apos;RECONNECT&apos;, lastSequenceNumber: 47 }
Server responds with REPLAY_EVENTS message containing events 48 through current
Client applies replayed events; canvas updates to current state
Connection status returns to green &apos;Connected&apos;
Reconnected user&apos;s cursor reappears for all other users

8. Accessibility Specifications
8.1 Non-Canvas Elements (WCAG 2.1 AA)
All interactive elements reachable via keyboard (Tab order is logical, follows visual layout)
Focus indicators: 2px blue outline (#2B6CB0) on all focusable elements
ARIA roles: role=&apos;toolbar&apos; on tool strip; role=&apos;log&apos; on Event Log panel (live region); role=&apos;list&apos; on Task Board; role=&apos;listitem&apos; on each task card
ARIA labels: aria-label on all icon-only buttons (e.g., aria-label=&apos;Add sticky note (N)&apos;)
Screen reader announcements: New task additions to Task Board announced via aria-live=&apos;polite&apos;
Connection status changes announced via aria-live=&apos;assertive&apos;

8.2 Color &amp; Contrast
All text elements: minimum 4.5:1 contrast ratio
AI tag badges: checked against backgrounds for contrast ≥ 4.5:1
Role indicators and lock icons: not solely color-dependent — include icon + text label
User cursor colors: colorblind-safe palette verified with simulation tools

8.3 Canvas Accessibility
[NOTE] The canvas drawing surface (HTML5 Canvas/SVG) has inherent accessibility limitations. The following mitigations apply:
Canvas nodes are mirrored in an off-screen ARIA tree for screen reader access
Keyboard users can Tab through canvas nodes when Select tool is active (Tab cycles through nodes by creation order)
Selected node details announced: &apos;Sticky note selected. Text: [content]. Author: [name]. Intent: ACTION_ITEM. Locked: No.&apos;

9. UX Rationale &amp; Design Decisions
Decision
Rationale
Alternative Considered
Floating pill toolbar (not edge toolbar)
Keeps canvas corners and edges free for content; pill is easily discoverable without consuming layout space
Left-edge icon rail — rejected because it competes with Event Log panel
AI badge on node (not separate list)
Intent is contextual — seeing the badge directly on the content it classifies eliminates context switching

Task Board on right (not bottom)
Vertical scroll of tasks is natural; bottom panels reduce canvas height which harms spatial thinking

Append-only event log (not editable history)
Builds user trust and supports time-travel replay; aligns with event-sourcing architecture
Mutable logs — rejected per BR-04 and architectural integrity
Lock icon bottom-left of node
Bottom-left is rarely used for content; top-right is occupied by AI badge; lock needs to be visible but not distracting
Top-left — rejected as it interferes with text editing handles
Cursor label pill (not tooltip)
Tooltip implies hover interaction; cursor labels are persistent presence indicators that need to be always visible without hover
Hover tooltip — rejected because presence must be ambient, not on-demand
Sticky note auto-resize vertically
Notes should never truncate content; truncation loses ideas
Fixed size with scroll — rejected as scroll on sticky note is unintuitive
